Select — native-parity styled dropdown select · v3 · NEW

<mythical-select> — a form-associated web component with native <select> parity: same attributes (name/value/disabled/required + form association, label focus, form reset, fieldset[disabled]), same API (value · selectedIndex · options · labels · validity · checkValidity() · reportValidity() · setCustomValidity()), input/change on user commit, groups via <mythical-optgroup label>, full keyboard incl. closed-typeahead. Reference implementation: packages/ui-core/src/select/mythical-select.js. Two authoring modes — pure tags, or progressive: wrap a real native <select>, which keeps working when JS is unavailable (what sandboxed previews show). variant="chip" gives the compact policy-chip look — squared, like every interactive control (§5). v2: SVG caret (rotates on open), ✓ on the current option, group separators, viewport-aware flip-up, reduced-motion-aware pop-in; engines without ElementInternals degrade to a working native select. Not supported: multiple/size — use a native select there.
