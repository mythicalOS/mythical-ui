# Support

## Where to ask

**Open an issue on the relevant repository.** Use the *Question* path in the issue chooser, or
the repository's Discussions if it has them enabled.

Before opening one, please check the repository's README and existing open and closed issues —
the answer is often already there.

## What to include

Questions that get answered quickly tend to include the repository and version, what you are
trying to achieve, what you tried, and what happened instead. Paste exact error output rather
than describing it.

## Response expectations

These projects are maintained by a very small team, in the open, on a best-effort basis. There
is no service-level agreement and no guaranteed response time on community support.

## Security issues

Do not use issues or discussions for vulnerabilities. Follow [SECURITY.md](SECURITY.md).

## Commercial support

Not offered yet.
